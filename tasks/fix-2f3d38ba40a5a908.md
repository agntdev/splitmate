# fix-2f3d38ba40a5a908 — E2T1 `/save` does not store timestamps; E2T2 `/list` sorts by name descending instead of timestamp descending

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 SPLT

`docs/spec.md` specifies that expenses must include an ISO-8601 timestamp and that `/list` outputs items in descending timestamp order (newest first). The current implementation stores only `{name, amount}` with no timestamp, and `MemoryListStore`/`RedisListStore` sort by **name** in reverse alphabetical order (`b.name.localeCompare(a.name)`) rather than by timestamp. The sorting logic is misleadingly labeled and diverges from the specification. Fix: store a timestamp in the save handler, sort by it in the list handler.

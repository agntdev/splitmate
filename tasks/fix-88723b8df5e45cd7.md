# fix-88723b8df5e45cd7 — parseFloat validation accepts partially-numeric strings (e.g. '10xyz')

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 SPLT

In `src/handlers/E1T1.ts:20`, `parseFloat(amountStr)` is used to validate the amount. `parseFloat` parses a numeric *prefix* — e.g. `parseFloat("10xyz")` returns `10`, and `isNaN(10)` is `false`, so the input passes validation and the bot splits 10 among users instead of rejecting the malformed input. This violates the task spec's "validation of inputs" requirement.

**Fix:** Use `Number(amountStr)` instead, which returns `NaN` for any non-fully-numeric string; or validate with a regex like `/^\d+(\.\d+)?$/`.

**Repro:** `/split 10xyz @alice @bob` → bot incorrectly responds with a split breakdown instead of an error.
